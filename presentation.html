Slide 02 — Introduction
Digital Footprints & Data Remanence
  • Every user action leaves a digital trace — files, logs, cache, metadata
  • "Deleting" a file only removes its directory entry, not its content
  • Actual data remains in storage until overwritten by new data
  • This is Data Remanence — persistence of data after deletion attempts
  • Forensic investigators exploit remanence to recover evidence from wiped devices
Key Concept — File System Abstraction
When a file is "deleted," the OS marks its directory entry as free.
The actual bytes on disk remain untouched until reallocated.
This forensic window can persist for days — or even years.
Average recovery rate after standard delete: ~95%
Common Misconceptions
✗ "Emptying recycle bin deletes my files"
✗ "Clearing browser history removes my data"
✗ "Formatting a drive destroys all data"
✗ "Cloud delete means it's gone forever"
Deletion Layer Model
USER ACTION — delete file.exe
↓

↓
PHYSICAL DISK BLOCKS — data still present ✓
Why This Matters

Even formatted drives retain recoverable fragments

Slide 03 — Problem Statement
The Problem With "Delete"
  • Standard deletion is a logical operation, not physical — the data bytes are never erased
  • Sensitive data — passwords, documents, browsing history — remains fully recoverable with free tools
  • Users and organizations maintain a false sense of security after clicking "Delete"
  • Improper device disposal is a leading cause of unintentional corporate data leaks and breaches
  • Legal and privacy violations (GDPR, HIPAA, PCI-DSS) result from unverified data destruction
User Deletes File
The OS removes the directory reference in milliseconds. No actual bytes are erased — the raw data sits intact on the physical disk sectors, waiting to be found by any forensic tool.
Investigator Scans Disk
Tools like Autopsy and Scalpel scan unallocated disk space, parse SQLite databases, and read raw sector data. Complete files are recovered within 3–8 seconds — no special hardware needed.
Privacy Breach
Recovered data exposes passwords, private documents, browser history, login sessions, and behavioral patterns — all retrieved from a device the owner believed was "wiped clean."

Slide 07 — Countermeasures
Secure Deletion — How It Works
  • Single-Pass Overwrite — writes zeros or random bytes over original data exactly once
  • Multi-Pass Wiping (DoD 5220.22-M) — overwrites 3–7 times with alternating patterns
  • Memory Sanitization — clears RAM buffers and swap/pagefile holding sensitive data
  • Cryptographic Erasure — AES-256 encrypts data then permanently discards the key
  • Verification — post-wipe scan confirms no original byte patterns remain on disk
Single-Pass: O(n) time, one write sweep — adequate for most personal data. Tools: sdelete (Win), srm (Linux)
DoD 5220.22-M: 7-pass military standard — alternating 0x00, 0xFF, random bytes. Required for classified disposal
Crypto Erase: AES-256 encrypt entire drive → destroy key instantly. Fastest method — the only reliable approach for SSDs
SSD Caveat — Wear Leveling
SSDs use wear leveling which may retain ghost copies in unaddressable cells.
Overwriting alone is insufficient for SSD secure deletion.
Cryptographic erasure is the only recommended SSD approach.

Slide 08 — Comparison
Normal vs Secure Deletion
  • Normal deletion: O(1) operation — only removes pointer, completes in milliseconds
  • Secure deletion: O(n) operation — must write to every byte of the target file
  • Recoverability after normal delete: ~95% of content recoverable via disk carving
  • Recoverability after multi-pass wipe: <1% — only unreadable header fragments remain
  • Trade-off: Security vs Speed — mitigated by selective wiping of sensitive files only
